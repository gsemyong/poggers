# Frozen Corpus Semantic Coverage

## Status

- Corpus: version 0.1
- Date: 2026-07-12
- Candidate: semantic operations
- Production acceptance: not implied

This matrix separates backend-independent semantic evidence from production web lowering. A case is
`covered` only when its required meaning can be composed from tested laws. `Partial` means at least
one observable semantic law remains absent. `Adapter-open` means the semantic language appears
adequate but real platform lowering and acceptance remain unproved.

## Cross-Corpus Result

The candidate has no demonstrated need for another styling value, behavior primitive, or
backend-shaped animation API. Every frozen case now composes from executable backend-independent
laws. This is semantic coverage, not production or usability acceptance: all cases remain
`adapter-open` until their real web lowering and empirical gates pass.

## Next Falsification Slice

Edge auto-scroll was selected first because it crosses recognizer intent, scroll ownership, geometry
revision, and continuous presentation without introducing a new visual target. The completed semantic
slice proves:

- only a structure-declared recognizer may drive a declared scroll container;
- velocity is zero outside the edge zones and monotonic toward a bounded maximum within them;
- logical direction works for either axis without physical left/right assumptions;
- scroll bounds clamp movement and stop the driver at the boundary;
- each applied scroll delta rebases the active gesture so the dragged identity does not jump;
- cancellation, capture loss, recognizer unavailability, and disposal stop auto-scroll immediately;
- preset parameters may tune edge extent and maximum speed but cannot choose semantic destinations;
- keyboard reorder/selection remains an accessible alternative and does not depend on auto-scroll.

The adjustable slice now proves one complete range (`minimum`, `maximum`, `step`, `largeStep`) and one
clamped, quantized resolver for pointer, keyboard, and direct numeric input. Maximum and minimum remain
reachable even when bounds do not lie on the step lattice. The HMR slice proves that presentation-only
replacement preserves context, state, surviving presence, and compatible presented motion samples,
while every old controller object is disposed once. Behavior, semantic role/action, or recognizer
contract changes require replacement.

The next falsification target is production web lowering: lower the candidate IR without authored
backend objects, then differentially verify native semantics, retained presentation, preset hot
replacement, and disposal against these reference laws.

Do not add an imperative animation loop, DOM scroll callback, CSS overflow option, or Anime.js object to
the author language. The reference law comes first; candidate syntax follows only if the law survives.
